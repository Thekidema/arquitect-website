DESIGN SYSTEM
================================================================================

COLOR PALETTE:

  LIGHT MODE:
    Primary Background:     #FAFAFA (warm white, not pure)
    Text Primary:          #1C1C1C (deep black)
    Text Secondary:        #6B7280 (medium gray)
    Accent:                #F97316 (bright orange)
    Surface:               #FFFFFF (pure white cards)
    Border:                #E5E5E5 (light gray)

  DARK MODE:
    Primary Background:     #0d0d0b (deep dark)
    Text Primary:          #f0ede6 (warm white)
    Text Secondary:        #8a8580 (muted tan)
    Accent:                #ff9d3d (soft orange)
    Surface:               #1a1a18 (dark cards)
    Border:                rgba(255, 189, 38, 0.1) (subtle gold)

TYPOGRAPHY:

  DISPLAY FONT:            Playfair Display (serif)
  BODY FONT:               Outfit (sans-serif)

  SIZING (use clamp() - NEVER fixed):
    H1:  clamp(2.5rem, 7vw + 0.5rem, 7.5rem)
    H2:  clamp(1.8rem, 4vw + 0.5rem, 4rem)
    H3:  clamp(1.4rem, 2.5vw + 0.5rem, 2.5rem)
    Body: clamp(1rem, 1.5vw + 0.3rem, 1.2rem)

  LINE-HEIGHT:
    Titles:    0.9 - 1.1
    Body text: 1.5 - 1.6

  LETTER-SPACING:
    Normal:    0
    Capitals:  0.08em - 0.15em

SPACING SYSTEM:

  Base Unit: 8px
  Scale: 8, 16, 24, 32, 48, 64, 96, 128px

  Use CSS Variables:
    --spacing-xs: 0.5rem (8px)
    --spacing-sm: 1rem (16px)
    --spacing-md: 1.5rem (24px)
    --spacing-lg: 2rem (32px)
    --spacing-xl: 3rem (48px)

  Responsive Gaps:
    clamp(1rem, 3vw, 3rem) for responsive spacing

EFFECTS LIBRARY:

  ✅ GLASSMORPHISM
     - Use: Header, cards, premium sections
     - CSS: backdrop-filter: blur(12px) saturate(180%)
     - Requires semi-transparent background
     - Webkit prefix for Safari support

  ✅ PARTICLE SYSTEM
     - Canvas-based 30-35 particles
     - Cursor tracking with lerp smoothing
     - Mobile optimization (disabled <768px)
     - Colors adapt to theme

  ✅ 3D TILT EFFECTS
     - Rotation: ±15° angles
     - Dynamic shadow scaling
     - Glare effect with customizable opacity
     - Smooth perspective transforms

  ✅ TEXT REVEALS
     - 4 types: clip-path, mask, fade, slide
     - Character/word/line granular control
     - IntersectionObserver triggers on scroll
     - Easing functions for smoothness

  ✅ SCROLL ANIMATIONS
     - Progress bar with glow
     - Animated counters sync with scroll
     - Parallax element support
     - Stagger reveal (80ms between items)

  ❌ DON'T USE
     - Blue-purple gradients (cliché)
     - Inter font as display (generic)
     - Emoji icons (unprofessional)
     - More than 5 different easing curves
     - Animated everything (use restraint!)

CODE ARCHITECTURE STANDARDS
================================================================================

JAVASCRIPT:

  GOOD PATTERN:

    class FeatureName {
      constructor() {
        this.init();
      }

      init() {
        this.setupEventListeners();
        this.render();
      }

      setupEventListeners() {
        element.addEventListener('click', () => this.handleClick());
      }

      handleClick() {
        // Logic here
      }

      cleanup() {
        element.removeEventListener('click', this.handleClick);
      }
    }

    window.featureName = new FeatureName();

  BAD PATTERNS:
  • Inline onclick handlers
  • Global variables pollution
  • No error handling
  • Silent failures
  • Memory leaks (no cleanup)
  • Hardcoded selectors (use data-* attributes)

  PERFORMANCE:
  • Use requestAnimationFrame for animations
  • Debounce scroll/resize listeners
  • Use transform/opacity only (GPU accelerated)
  • Guard clauses for null checks
  • Lazy load non-critical modules

CSS:

  ORGANIZATION:

    1. Design Tokens
       :root { --color-bg: #FAFAFA; }

    2. Layout Utilities
       .container { max-width: 1280px; }

    3. Components
       .button { /* styles */ }

    4. Animations
       @keyframes fadeUp { /* keyframes */ }

    5. Utilities
       .text-center { text-align: center; }

  PREMIUM EFFECTS:

    Glassmorphism:
      background: rgba(255, 255, 255, 0.85);
      backdrop-filter: blur(12px) saturate(180%);
      -webkit-backdrop-filter: blur(12px) saturate(180%);
      border: 1px solid rgba(249, 115, 22, 0.15);
      box-shadow: 0 4px 30px rgba(0, 0, 0, 0.1);

    Gradient Text:
      background: linear-gradient(135deg, #F97316 0%, #ff9d3d 100%);
      -webkit-background-clip: text;
      -webkit-text-fill-color: transparent;

    Magnetic Button:
      .button::before {
        transform: scaleX(0);
        transform-origin: left;
        transition: transform 0.35s cubic-bezier(0.76, 0, 0.24, 1);
      }
      .button:hover::before { transform: scaleX(1); }

  ANTI-PATTERNS:
  • No generic box-shadow (use specific values)
  • No !important (unless CSS reset)
  • No fixed widths (use max-width + fluid)
  • No border-radius: 8px everywhere (define per component)
  • No hardcoded colors outside variables.css

HTML:

  SEMANTIC STRUCTURE:

    <header class="header-premium">
      <nav aria-label="Primary Navigation">
        <a href="#" class="nav-link active" aria-current="page">Home</a>
      </nav>
    </header>

    <main>
      <section id="heroes">
        <h1>Hero Title</h1>
      </section>
    </main>

    <footer>
      <p>&copy; 2026 ARQUITECT</p>
    </footer>

  ACCESSIBILITY:
  • ARIA labels on buttons/toggles
  • Role attributes on custom elements
  • Proper heading hierarchy (h1 > h2 > h3)
  • Alt text on meaningful images
  • Focus states visible
  • Semantic HTML (not divitis)

ANIMATION & INTERACTION STANDARDS
================================================================================

ANIMATION HIERARCHY:

  1. PAGE LOAD (200-300ms)
     - Most important first impression
     - Hero fade-in, title reveal

  2. SCROLL REVEALS (80ms stagger)
     - Intersection Observer with delay
     - Content appears as user scrolls

  3. HOVER STATES (300-400ms)
     - Buttons scale/change color
     - Cards elevate with shadow

  4. CURSOR EFFECTS (Optional)
     - Particle following
     - Only if theme supports

  5. STATE TRANSITIONS (400-600ms)
     - Modal opens/closes
     - Theme toggles
     - Tab switches

IMPLEMENTATION:

  IntersectionObserver Pattern:

    const observer = new IntersectionObserver((entries) => {
      entries.forEach((entry, i) => {
        if (entry.isIntersecting) {
          setTimeout(() => {
            entry.target.classList.add('is-visible');
          }, i * 80); // 80ms stagger
          observer.unobserve(entry.target);
        }
      });
    }, { threshold: 0.15, rootMargin: '0px 0px -60px 0px' });

    document.querySelectorAll('[data-reveal]').forEach(el => observer.observe(el));

PERFORMANCE GUIDELINES:

  ✅ Use transform & opacity ONLY for animations
  ✅ Use requestAnimationFrame for smooth 60fps
  ✅ Set will-change: transform ONLY during animation
  ✅ Disable particles on mobile (<768px)
  ✅ Reduce animation complexity on slow devices
  ✅ Debounce scroll listeners

  ❌ DON'T animate: top, left, width, height (layout thrashing)
  ❌ DON'T use setTimeout for animations (jank)
  ❌ DON'T add will-change to everything (memory waste)
  ❌ DON'T skip cleanup (memory leaks)

EASING FUNCTIONS:

  Standard:   cubic-bezier(0.25, 0.1, 0.25, 1.0)
  Snappy:     cubic-bezier(0.34, 1.56, 0.64, 1)
  Smooth:     cubic-bezier(0.4, 0, 0.2, 1)
  Bounce:     cubic-bezier(0.68, -0.55, 0.265, 1.55)

TIMING:

  Reveal animations:   200-300ms
  Hover effects:       300-400ms
  Page transitions:    400-600ms
  Stagger delay:       80-100ms between items
  Dark mode toggle:    300ms smooth transition

RESPONSIVE DESIGN STANDARDS
================================================================================

MOBILE-FIRST APPROACH:

  Start with 320px design, enhance for larger screens

  Breakpoints:
    360px   (small phones)
    768px   (tablets)
    1024px  (desktops)
    1440px  (large screens)

FLUID TYPOGRAPHY:

  NO FIXED SIZES!

  h1 { font-size: clamp(2.5rem, 7vw + 0.5rem, 7.5rem); }
  h2 { font-size: clamp(1.8rem, 4vw + 0.5rem, 4rem); }
  h3 { font-size: clamp(1.4rem, 2.5vw + 0.5rem, 2.5rem); }
  p  { font-size: clamp(1rem, 1.5vw + 0.3rem, 1.2rem); }

RESPONSIVE GRIDS:

  Desktop (3 columns):
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: clamp(1rem, 3vw, 3rem);

  Tablet (2 columns):
    @media (max-width: 1024px) {
      grid-template-columns: repeat(2, 1fr);
    }

  Mobile (1 column):
    @media (max-width: 768px) {
      grid-template-columns: 1fr;
    }

TOUCH TARGETS:

  Minimum: 44px × 44px
  Spacing: 16px between targets

  Good:
    Buttons, links, interactive elements ≥ 44px

  Bad:
    < 44px touch targets (hard to hit on mobile)
    Crowded elements (no spacing)

IMAGES:

  Responsive Images:
    <picture>
      <source srcset="image.webp" type="image/webp">
      <img src="image.jpg" alt="Description" loading="lazy">
    </picture>

  Avoid:
    Fixed widths
    Large file sizes
    Missing alt text

DECISION TREE FOR COMMON ISSUES
================================================================================

Q: "Images not loading"
A: Check paths from public/index.html
   Wrong: src/assets/images/hero/img.svg
   Right: ../src/assets/images/hero/img.svg

Q: "CSS not applied"
A: Verify <link rel="stylesheet"> in HTML head
   Check file exists at path
   Clear browser cache (Ctrl+Shift+R)

Q: "Dark mode broken"
A: Check browser allows localStorage
   Verify dark-mode.js loaded
   Check console for errors

Q: "Animations janky"
A: Use transform & opacity ONLY
   Never animate top, left, width, height
   Profile with DevTools Performance tab
   Reduce particle count on mobile

Q: "3D tilt not working"
A: Verify enhanced-tilt.js loaded
   Check elements have correct classes
   Test in DevTools console: window.VanillaTilt

Q: "Particles invisible"
A: Mobile (<768px) disables particles
   Check GPU acceleration enabled
   Verify canvas element rendering
   Check console for errors
